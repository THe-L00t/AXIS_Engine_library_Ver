# AXIS Library

AXIS Library는 **시간(Time), 공간(Space), 데이터(Data)**를 명확한 축으로 정렬하는
라이브러리 기반 게임 엔진 코어 프로젝트입니다.

이 저장소는 완성된 상용 엔진이 아니라,
엔진을 구성하는 **구조·데이터 흐름·성능 특성**을 검증하고 축적하기 위한
AXIS Engine의 기반 레이어입니다.

---

## 프로젝트 개요

* 프로젝트명: **AXIS Library**
* 형태: C++ 기반 DLL 라이브러리 집합
* 개발 환경:

  * Windows 11
  * Visual Studio 2022
* 라이선스: **Apache License 2.0**
* 목적:

  * 게임 엔진의 핵심 구조를 라이브러리 레벨에서 명확히 정의
  * 엔진 이전 단계에서 설계 철학과 성능 특성을 검증
  * 이후 AXIS Engine (에디터/툴링 포함)의 기반으로 사용

AXIS Library만으로도 **완성된 게임을 만드는 것이 가능**해야 하며,
AXIS Engine은 이 라이브러리 위에 GUI와 에디터를 덧씌운 다른 표현 형식에 가깝습니다.

---

## AXIS의 철학

AXIS는 기능의 집합이 아니라,
**물리적으로 이해 가능한 기준 위에 정렬된 시스템**을 지향합니다.

### Three Axes

* **Time**
  언제 무엇이 실행되는가
  → 명시적인 업데이트 순서, 예측 가능한 실행 흐름

* **Space**
  무엇이 어디에 존재하는가
  → 구조적으로 드러나는 소속과 범위

* **Data**
  무엇이 메모리에 존재하며 어떻게 흐르는가
  → 숨겨지지 않는 데이터 생명주기

이 세 축은 기능 분류가 아니라,
모든 설계 판단의 기준 좌표계로 사용됩니다.

---

## AXIS Library가 하지 않는 것

AXIS는 구조적 일관성을 유지하기 위해 다음을 **의도적으로 제공하지 않습니다**.

* 특정 장르에 특화된 전용 시스템
* 자동 최적화 및 블랙박스 처리
* 엔진 내부 동작을 숨기는 편의 기능
* 에디터 중심의 개발 강제
* 즉각적인 시각적 결과를 보장하는 Core 기능

AXIS는 “바로 보이게 만드는 엔진”이 아니라,
“올바르게 설계하면 예측 가능한 결과를 내는 기반”을 목표로 합니다.

---

## 성능에 대한 관점

AXIS에서 성능은 결과가 아니라 **설계 대상**입니다.

* 성능 비용은 숨겨지지 않아야 합니다.
* 동일한 구조는 항상 유사한 성능 특성을 가져야 합니다.
* 자동 최적화보다 명시적인 구조 선택을 우선합니다.

엔진은 결정을 대신 내려주지 않고,
개발자가 성능을 **이해하고 통제할 수 있는 구조**를 제공합니다.

---

## 라이브러리와 엔진의 관계

AXIS 프로젝트는 두 가지 형식으로 존재합니다.

### AXIS Library

* 순수 코드 기반 사용
* 최소 의존성
* 명확한 API
* 구조 실험 및 게임 제작 가능

### AXIS Engine (예정)

* AXIS Library 위에 구축
* 일부 코드 + 일부 GUI 기반 워크플로우
* 에디터, 시각화 도구 제공
* 상용화 수준 목표

두 버전은 별개의 제품이 아니라,
**하나의 철학을 공유하는 서로 다른 층위의 결과물**입니다.

## 대상 사용자

AXIS Library는 모든 개발자를 위한 프로젝트가 아닙니다.

이 프로젝트는 다음과 같은 개발자를 대상으로 합니다.

* 엔진의 내부 구조를 이해하고 싶은 개발자
* 성능과 시스템을 직접 설계하고 싶은 개발자
* 자동화보다 통제를 선호하는 개발자
* 라이브러리 기반 개발에 익숙하거나 관심 있는 개발자
